# TDD Pipeline Master Prompt for Claude Code

## 📋 使用方式

将以下模板的 `<...>` 替换为你的具体需求，然后执行：

```bash
claude code --path tdd-toolkit/ --prompt "[TDD Pipeline] <完整 Prompt>"
```

---

## 🎯 完整 Prompt 模板

```
[TDD Pipeline] 请帮我实现 <我的功能名称>，使用强约束 TDD 流水线，按顺序完成每个节点

## 项目信息
- 项目名称：<project-name>
- 功能描述：<详细描述要开发的功能>
- 预期输入：...
- 预期输出：...

## 开发规则（必须遵守）

### 1. 任务规划阶段（Planning）
- [ ] 明确单一功能点边界
- [ ] 画出核心流程图或伪代码
- [ ] 识别边界条件和异常场景
- [ ] 记录到：`tdd-sessions/<project-name>-tasks.md`
- [ ] **强制执行**：完成此阶段才能进入下一步

### 2. 单元测试（Unit Test）
- [ ] 创建测试文件（tests/xxx_test.py）
- [ ] 编写至少 3 个测试用例：
  - 正常路径测试
  - 异常路径测试  
  - 边界条件测试
- [ ] **首次运行测试**（预期会失败）
- [ ] 记录到：`tdd-sessions/tasks/unit-test.md`
- [ ] **强制执行**：测试失败是预期的，不要跳过

### 3. 代码实现（Code Implementation）
- [ ] 读取测试用例，理解预期行为
- [ ] 编写实现代码（src/xxx.py）
- [ ] 确保代码能解析/编译
- [ ] **强制执行**：先确保语法正确

### 4. 编译/检查（Compile/Check）
- [ ] `pytest --collect-only` 检查语法
- [ ] `mypy` 类型检查（如有）
- [ ] `black --check` 风格检查
- [ ] **强制执行**：通过所有检查才能继续

### 5. 运行测试（Run Tests）
- [ ] `pytest -v tests/`
- [ ] 检查覆盖率
- [ ] **强制执行**：所有测试必须通过

### 6. 调试回溯（Debug if Failed）
- [ ] 如果测试失败，分析错误日志
- [ ] 检查测试用例是否正确
- [ ] 检查实现代码的 bug
- [ ] **调用 IssueTracker** 记录问题到 `pit-library/`
- [ ] 修复后重新测试（循环直到全部通过）
- [ ] 记录到：`tdd-sessions/tasks/debug.md`
- [ ] **强制执行**：有问题必须记录

### 7. 完成（Complete）
- [ ] 所有测试通过且稳定
- [ ] 代码审查（self-review）
- [ ] 更新 MEMORY.md
- [ ] 标记流水线完成
- [ ] **强制执行**：最后一步

## 避坑检查

请在每个阶段开始前检查 `pit-library/`，如果有相似问题：
- v001: <已知的类似问题> → 解决方案：<简述>

## 强制约束

1. **每个节点必须完成才能进入下一步**（检查清单机制）
2. **测试必须先写**，实现代码是第二步
3. **失败的测试是预期的**，正确是"先失败后通过"
4. **所有问题必须记录**到 pit-library（作为自我提升）
5. **每一步都有文档**（在 tdd-sessions/）

## 输出要求

开发完成后，生成一份完整的报告，包含：

1. **Pipeline 状态**：当前在哪个节点，是否完成
2. **任务列表**：每个节点的执行结果
3. **测试报告**：测试文件位置、覆盖率、结果
4. **问题记录**：已记录的问题 ID 和解决方案
5. **下一步建议**：如果有待优化点

## 现在开始

请按顺序开启 TDD Pipeline，从任务规划开始。
```

## 💡 进阶用法

### 方式 1: TDD Pipeline + 当前项目

```bash
# 在 Claude Code 里直接调用
claude code --path /Users/dylan/.openclaw/workspace/tdd-toolkit/ \
  --prompt "[TDD Pipeline] 帮我测试 webhook 审批功能，项目名：webhook-approval"
```

### 方式 2: 手动推进 Pipeline

```bash
# 1. 先初始化 Pipeline
python tdd-pipeline.py webhook-approval planning

# 2. 让 Claude Code 写测试
claude code --prompt "根据规划，在 tests/ 创建 test_approval.py"

# 3. 返回 Pipeline 工具，推进到下一步
python tdd-pipeline.py webhook-approval unit_test

# 4. 循环直到完成
```

### 方式 3: 故障回溯

```bash
# 测试失败时
python issue-tracker.py record \
  --feature "webhook-approval" \
  --error-type "AssertionError" \
  --message "审批逻辑不正确" \
  --pipeline-node "run_tests"
```
