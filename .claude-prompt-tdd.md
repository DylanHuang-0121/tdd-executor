# TDD 开发任务 Prompt 模板

## 使用说明

将以下模板的 `<...>` 替换为你的具体需求，然后传给 Claude Code：

```
claude code --path tdd-toolkit/ --prompt "[TDD 开发任务] 请帮我实现 <功能名称>，使用测试驱动开发模式..."
```

---

## 完整 Prompt 模板

```
[TDD 开发任务] 请帮我实现 <我的功能名称>，使用测试驱动开发模式

## 需求描述
<详细描述你要实现的功能>

## 开发规则
1. 使用 pytest 框架
2. 先写测试，再写实现
3. 确保测试通过后再提交
4. 如果发现潜在问题，记录到 pit-library/
5. 遵循最小可运行原则，每次只实现一个功能点

## 测试要求
- 测试文件名：test_<功能名称>.py
- 测试目录：tests/
- 测试边界情况：包括但不限于空输入、极端值、异常输入

## 避坑检查
请检查 pit-library/ 中是否有类似问题，如果有，请在开发时特别注意

## 输出
开发完成后，请生成一份测试报告，包含：
- 测试文件位置
- 测试结果
- 发现的问题及解决方案（如果有）
```

---

## 运行流程

1. **💡 调用 Prompt**
   ```bash
   claude code --path tdd-toolkit/ --prompt "[上面的模板]"
   ```

2. **🤖 Claude Code 执行**
   - 检查 pit-library 避坑
   - 生成测试文件
   - 运行测试（第一次失败是预期的）
   - 生成实现代码
   - 重新运行测试（应该通过）

3. **📝 问题记录**
   - 如果测试失败，Claude Code 调用 `issue-tracker.py record`
   - 问题自动写入 pit-library/

4. **✅ 输出报告**
   - 测试覆盖情况
   - 已解决的问题
   - 待关注点

## 进阶用法

### 方式 1：配合 TDD Runner

```bash
# 1. Claude Code 帮你生成测试和实现
claude code --path project/ --prompt "帮我测试 webhook 审批功能"

# 2. 用 tdd-runner 验证和记录问题
python tdd-runner.py webhook-approval --test-dir tests/
```

### 方式 2：手动补充 pit-library

```bash
# 如果你在开发中遇到问题，手动记录
python issue-tracker.py manual
```
